# 项目简介
这是一个面向小说创作的工作台，包含大纲生成与编辑、角色关系图、版本快照与恢复、写作润色与一致性分析，以及面向仿写学习的文笔知识库等功能。前端使用 Next.js，后端使用 FastAPI，数据存储在本地 `backend/data/`。

**功能一览**
- AI 生成大纲  
用途：根据世界观、风格标签和核心创意快速生成完整故事结构。  
使用方式：`新建大纲 -> AI 生成`，填写世界观与核心创意后提交。
- 空白项目  
用途：手动规划大纲，不依赖 AI。  
使用方式：`新建大纲 -> 空白大纲`，填写项目名称后创建。
- 大纲视图（节点图 + 列表）  
用途：用“节点”组织剧情，并支持时间轴与叙事顺序。  
使用方式：点击节点打开编辑器，新增/插入节点，拖动重排，导入文本大纲。
- 节点同步与冲突检测  
用途：保存节点时自动更新知识图谱与索引，并提示设定冲突。  
使用方式：在节点编辑器中修改并保存，右下角会提示冲突。
- 角色关系图  
用途：可视化角色网络，支持新增角色/关系、合并与同步。  
使用方式：`关系`页面查看图谱并编辑，必要时点击“同步图谱”。
- 大纲一致性分析  
用途：基于大纲检索、角色关系图与冲突检测，输出一致性问题与修改建议。  
使用方式：进入 `/analysis` 页面进行一致性分析与修改建议审查。
- 写作工作台  
用途：管理章节内容并提供 AI 润色/扩写。  
使用方式：`写作`页面创建章节、编辑正文，选择“全文/选中段落润色/扩写”。
- 仿写文笔知识库  
用途：导入其他小说文本作为写作风格参考；系统会先按批次交给 AI 清洗，再对清洗结果做 RAG 检索。  
使用方式：`写作`页面右侧 AI 面板上传 `.txt/.md` 小说文本，勾选要参考的文笔素材。
- 版本快照与恢复  
用途：保存关键版本、查看差异、随时回滚。  
使用方式：点击左侧“版本历史”，或 `Ctrl/Cmd + S` 创建手动快照，`Ctrl/Cmd + Shift + H` 打开历史面板。
- 导出/导入项目  
用途：跨环境迁移项目与备份。  
使用方式：项目列表中导出/导入 JSON。

**环境要求**
- Node.js >= 18
- Python >= 3.11
- pnpm
- Poetry

**环境变量**
- `OPENAI_API_KEY`：默认 LLM API Key
- `OPENAI_BASE_URL`：LLM Base URL
- `MODEL_NAME`：默认模型名
- `OPENAI_API_KEY_DRAFTING`：大纲生成专用 Key（可选）
- `OPENAI_API_KEY_SYNC`：同步分析专用 Key（可选）
- `OPENAI_API_KEY_EXTRACTION`：实体抽取专用 Key（可选）
- `MODEL_NAME_DRAFTING`：大纲生成模型名（可选）
- `MODEL_NAME_SYNC`：同步分析模型名（可选）
- `MODEL_NAME_EXTRACTION`：实体抽取模型名（可选）
- `EMBEDDING_MODEL`：向量嵌入模型（可选，默认 `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2`）
- `CHROMA_PERSIST_PATH`：向量库持久化目录（默认 `backend/data/chroma_db`）
- `CORS_ALLOW_ORIGINS`：允许的前端域名，逗号分隔
- `NEXT_PUBLIC_API_URL`：前端请求后端的地址（默认 `http://localhost:8000`）

**项目结构**
- 前端：`frontend/`（Next.js、React）
- 后端：`backend/`（FastAPI、LangGraph、向量检索）
- 数据：`backend/data/`（SQLite、向量库、知识图谱、版本快照）
- 配置：`.env.example`、`.gitignore`

**快速上手**
1. 安装依赖（前端）：`pnpm install`
2. 安装依赖（后端）：`poetry install`
3. 根据 `.env.example` 创建本地 `.env`
4. 启动后端：
   `cd backend`
   `poetry run uvicorn app.main:app --host 0.0.0.0 --port 8000`
5. 启动前端：
   `cd frontend`
   `pnpm dev -H 0.0.0.0 -p 3000`
6. 访问：`http://localhost:3000`

**Docker 启动**
生产模式（构建镜像并运行）：
1. 创建 `.env`（至少包含 `OPENAI_API_KEY`）
2. 启动：`docker compose up --build`
3. 停止：`docker compose down`
4. 访问：
   `http://localhost:8000`（后端）
   `http://localhost:3000`（前端）

开发模式（热重载）：
1. 启动：`docker compose -f docker-compose.dev.yml up --build`
2. 停止：`docker compose -f docker-compose.dev.yml down`
3. 前端容器内会执行 `pnpm dev`，后端使用 `uvicorn --reload`

**使用流程（推荐）**
1. `新建大纲`：选择 AI 生成或空白项目。
2. `大纲`：在节点图中补充剧情、调整叙事顺序与时间轴位置。
3. `关系`：查看角色网络，必要时补充角色与关系。
4. `写作`：创建章节，上传参考小说构建文笔知识库，再使用 AI 润色/扩写提升表现力。
5. `分析`：在 `/analysis` 页面做一致性检查与修改建议。
6. `版本历史`：关键节点创建快照，必要时回滚。

**功能详解**
**大纲导入**
用途：将已有大纲文本解析为节点、角色与世界观。  
使用方式：`大纲`页面点击导入，粘贴文本或上传 `.md/.txt`。

**节点编辑与同步**
用途：编辑节点内容、时间轴与角色信息，并触发图谱/索引更新。  
使用方式：点击节点打开编辑器，保存后自动同步；冲突会在右下角提示。

**仿写文笔知识库**
用途：导入参考小说，先由 AI 分批清洗出更适合学习语感、节奏、描写方式的片段，再用于 RAG 检索与风格参考。  
使用方式：`写作`页面右侧 AI 面板上传 `.txt/.md` 小说文本，系统会自动清洗、切分并入库；支持预览/重命名/删除。

**角色关系图**
用途：呈现角色之间的关系网络，并与大纲节点关联。  
使用方式：`关系`页面选择节点筛选角色、添加关系，必要时执行图谱同步。

**大纲分析**
用途：基于大纲、角色关系图与历史分析记录做一致性分析。  
使用方式：进入 `/analysis`，输入问题或点击“快速分析”。

**写作与润色**
用途：章节化写作并支持 AI 润色/扩写。  
使用方式：`写作`页面创建章节，编辑正文，选择“全文/选中段落润色/扩写”。可在写作助手配置中设置“润色提示词/扩写提示词/风格强度/风格预设”。

**版本快照**
用途：保存版本、查看差异与回滚。  
使用方式：左侧“版本历史”查看列表；快捷键 `Ctrl/Cmd + S` 创建手动快照。当前默认只支持手动快照，旧项目中的其他快照类型会作为历史数据兼容显示。

**模型与 Prompt 配置**
用途：为不同任务设置模型与 API Key，或覆盖默认 Prompt。  
使用方式：左侧“设置”配置模型与 Key，“P”按钮编辑 Prompt；写作面板内可配置润色/扩写提示词与风格强度。

**RAG 检索策略**
- 默认混合检索：向量 + 关键词 + BM25，并进行加权融合（0.6/0.2/0.2）。
- 适用范围：大纲分析检索、仿写文笔知识库检索。
- 文笔知识库清洗目标：去除版权页、章节号噪声、无关说明与低信息重复段，保留更利于学习句式、节奏、叙述视角、描写技法的片段。

**导出/导入项目**
用途：导出完整项目数据（大纲、知识图谱、文笔知识库与快照）。  
使用方式：项目列表点击“导出/导入”。注意：导入会保留原项目 ID。

**本地部署提示**
- 当前默认无鉴权与权限隔离，仅建议在本地或内网使用。
- 数据默认存放在 `backend/data/`，包含 `stories.db`、向量库与快照。
- 建议定期备份 `backend/data/`。
